## 🚀 DEPLOYMENT INSTRUCTIONS (Step-by-Step)

### PHASE 1: LOCAL SETUP (10 minutes)

**Step 1: Install Prerequisites**
```bash
# Ensure Ruby is installed (version 3.0+)
ruby --version

# Install Bundler
gem install bundler
```

**Step 2: Navigate to Project**
```bash
cd "c:\Users\User PC\Documents\degreartcode data\BlogUsage\LatestSearchAI\wesearchai"
```

**Step 3: Install Dependencies**
```bash
bundle install
```

**Step 4: Configure Site (_config.yml)**
Edit `_config.yml` and update:
```yaml
url: "https://yourusername.github.io/wesearchai"
title: "WeSearchAI"
description: "Your trusted source for global news..."
author: "Your Name"
email: "your-email@example.com"
```

**Step 5: Add Google AdSense Codes**
- Get your AdSense Publisher ID (format: `ca-pub-1234567890123456`)
- Replace in `_layouts/default.html` line ~25
- Replace in `_layouts/post.html` line ~10 and ~45
- Update `ads.txt` with your ID

**Step 6: Test Locally**
```bash
bundle exec jekyll serve
# Visit http://localhost:4000
```

### PHASE 2: GITHUB SETUP (5 minutes)

**Step 1: Create GitHub Repository**
1. Go to [GitHub.com](https://github.com)
2. Click "New Repository"
3. Name it: `yourusername.github.io` (use your actual GitHub username)
4. Make it public
5. Click "Create Repository"

**Step 2: Clone Repository**
```bash
git clone https://github.com/yourusername/yourusername.github.io.git
cd yourusername.github.io
```

**Step 3: Copy WeSearchAI Files**
- Copy all files from `wesearchai/` folder into the cloned repository
- Ensure hidden files like `.gitignore` are included

**Step 4: Initialize Git (if needed)**
```bash
git init
git add .
git commit -m "Initial commit: Deploy WeSearchAI blog"
git branch -M main
git remote add origin https://github.com/yourusername/yourusername.github.io.git
git push -u origin main
```

### PHASE 3: GITHUB PAGES ACTIVATION (2 minutes)

1. Go to your repository on GitHub
2. Click **Settings** (top right)
3. Left sidebar: Click **Pages**
4. **Build and deployment** section:
   - Source: Select "Deploy from a branch"
   - Branch: Select "main"
   - Folder: Select "/ (root)"
5. Click **Save**
6. Wait 1-2 minutes for deployment
7. Visit: `https://yourusername.github.io`

### PHASE 4: CUSTOM DOMAIN (Optional, 5 minutes)

1. Purchase domain (GoDaddy, Namecheap, etc.)
2. In GitHub repository Settings > Pages
3. Enter custom domain (e.g., `yoursite.com`)
4. In your domain provider, add DNS records:
   ```
   A record pointing to 185.199.108.153
   A record pointing to 185.199.109.153
   A record pointing to 185.199.110.153
   A record pointing to 185.199.111.153
   CNAME record pointing to yourusername.github.io
   ```
5. Enable HTTPS (GitHub automatically handles this)

### PHASE 5: GOOGLE ADSENSE APPROVAL (7-15 days)

1. Wait 30+ days after purchasing/launching domain (for indexing)
2. Visit [Google AdSense](https://www.google.com/adsense)
3. Click "Sign up now"
4. Enter your website URL
5. Complete verification process:
   - Add verification code to your site
   - Wait for Google to verify
6. Once approved:
   - Update AdSense codes in your site
   - Enable ads on your pages
7. Start earning!

## 📞 TROUBLESHOOTING

### Issue: "jekyll command not found"
**Solution**: Run `bundle install` then prefix commands with `bundle exec`

### Issue: Posts not showing
**Solution**: 
- Check filename format: `YYYY-MM-DD-title.md`
- Verify front matter syntax (YAML)
- Run `bundle exec jekyll clean` then rebuild

### Issue: Site won't build
**Solution**:
```bash
bundle install --redownload
bundle exec jekyll build
```

### Issue: GitHub Pages not updating
**Solution**:
1. Wait 5 minutes
2. Check Settings > Pages > Build & deployment
3. Check build logs (Actions tab)
4. Verify branch is set to "main"

### Issue: AdSense code not appearing
**Solution**:
- Verify you replaced ALL placeholder IDs
- Clear browser cache (Ctrl+Shift+Delete)
- Wait 24-48 hours for AdSense approval
- Check browser console for errors (F12)
